## 自動撮影スクリプトの使用方法

スクリーンショットを自動的に撮影するスクリプトが用意されています。

### 前提条件

1. 開発サーバーが起動していること
2. データベースにシードデータが投入されていること
3. 管理者ユーザー（admin/admin123）が存在すること

### 実行手順

```bash
# 1. 開発サーバーを起動（別のターミナルで）
npm run dev

# 2. サーバーが起動するまで待つ（http://localhost:3000 にアクセスできることを確認）

# 3. スクリーンショット撮影スクリプトを実行
npx tsx scripts/capture-screenshots.ts
```

## 手動撮影の方法

自動スクリプトがうまく動作しない場合は、手動で撮影してください。

### 手順

1. ブラウザで http://localhost:3000 にアクセス
2. 各画面に遷移
3. ブラウザの開発者ツール（F12）でビューポートを 1920x1080 に設定
4. スクリーンショットを撮影（Windows: Win+Shift+S、Mac: Cmd+Shift+4）
5. 上記のファイル名で保存

### 撮影のポイント

- **解像度**: 1920x1080 推奨
- **ブラウザ**: Chrome推奨
- **データ**: サンプルデータが表示されている状態で撮影
- **UI**: メニューやダイアログは閉じた状態で撮影（特に指定がない場合）
- **フルページ**: 長いページは全体を撮影（スクロール必要）

## トラブルシューティング

### スクリプトが失敗する場合

**エラー: Timeout 30000ms exceeded**

- 開発サーバーが起動していることを確認
- シードデータが投入されていることを確認
- ログイン情報（admin/admin123）が正しいことを確認

**エラー: Browser not found**

```bash
# Playwrightブラウザを再インストール
npx playwright install chromium
```

### 画像が生成されない場合

- `docs/screenshots/` ディレクトリが存在することを確認
- ディレクトリの書き込み権限を確認

### 画像が真っ白になる場合

- ページの読み込みが完了する前に撮影されている可能性があります
- スクリプトの `waitForLoadState('networkidle')` の後に `page.waitForTimeout(1000)` を追加してください
